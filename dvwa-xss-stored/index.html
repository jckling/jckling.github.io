DVWA Writeup Part XI (XSS (Stored))

发表于2020-04-26|tech
|浏览量:|评论数:

XSS (Stored)

存储型 XSS

Low

源码解析

从 $_POST 中获取数据，使用 trim 去除首位空白字符；用 stripslashes 对消息 Message 进行反引用，再使用 mysqli_real_escape_string 转义名称 Name 和消息 Message 中的 SQL 特殊字符；最后将数据插入数据库中。

漏洞利用

并没有关于 XSS 的防护，直接在 Message 一栏里注入代码；

之后每次打开这个页面都会弹窗；

先重置数据库，然后在 Name 字段注入相同代码，发现有长度限制，打开浏览器开发者工具，修改网页源码绕过限制；

具体位置如下，这里最大长度改为 99 。

参阅

  • PHP Manual - $_POST
  • PHP Manual - trim
  • PHP Manual - stripslashes
  • PHP Manual - mysqli_real_escape_string

Medium

源码解析

改用 addslashes 对 Message 进行处理，并用 strip_tags 去除 HTML 和 PHP 标记；使用 htmlspecialchars 将其中的特殊字符转换为 HTML 实体；使用 str_replace 将 Name 中的 <script> 进行替换。

漏洞利用

先重置数据库，同样使用开发者工具绕过限制，采用拼接的方式在 Name 字段中进行注入;

注入成功。

参阅

  • PHP Manual - addslashes
  • PHP Manual - strip_tags
  • PHP Manual - stripslashes
  • PHP Manual - mysqli_real_escape_string

High

源码解析

在 Medium 级别的基础上，把 str_replace 改为 preg_replace 对 Name 中的字符进行正则匹配和替换。

漏洞利用

还是先重置数据库，换一种 HTML 标签进行注入，仍然是在 Name 中进行注入，同样的方法绕过长度限制；

成功弹框。

参阅

  • PHP Manual - preg_replace

Impossioble

源码解析

使用 Anti-CSRF Token 来避免 CSRF 攻击，提交后先对 token 进行检查；对 Message 和 Name 进行反引用，转义 SQL 中使用的特殊字符，然后再将 & " ' < >转换为 HTML 实体；最后使用预处理的 SQL 语句执行数据的插入。
