DVWA Writeup Part III (CSRF)

发表于2020-04-23|Security
|阅读量:|评论数:

CSRF

跨站请求伪造

重置 DVWA 数据库

忘记登陆密码时（是我），打开页面 http://192.168.56.102/setup.php，点击 Create / Reset Database 重置数据库。

默认用户名 admin ，密码 password 。

Low

源码解析

直接用 $_GET 从 URL 中获取参数，检查两次输入的密码是否相同，如果相同则修改密码。

漏洞利用

直接构造 http://192.168.56.102/vulnerabilities/csrf/?password_new=123456&password_conf=132456&Change=Change# ，只要诱骗用户在当前 DVWA 的会话中点击链接即可实现攻击。

Medium

源码解析

使用 stripos 从 $_SERVER[ 'HTTP_REFERER' ] 中查找 $_SERVER[ 'SERVER_NAME' ]，即检查 HTTP Header 中的 Host 字段是否出现在 Refer 字段（在同一个域名下）；然后就是检查两次输入的密码是否相同，相同则修改密码。

漏洞利用

和 Low 级别的利用方法相同，将伪造页面命名为包含服务器域名的名称即可。

参阅

  • PHP Manual - stripos
  • PHP Manual - $_SERVER

High

源码解析

使用 Anti-CSRF Token 来避免 CSRF 攻击，提交数据后先对 token 进行检查；然后确认两次输入的密码是否相同，相同则修改密码。

漏洞利用

浏览器中查看网页源码，url 中是上一次修改密码时提交的 user_token ，源码中隐藏着本次需要提交的 user_token。

Impossioble

源码解析

首先需要输入原始密码，然后输入两次修改的密码；提交数据后先对 token 进行检查,然后使用 stripslashes 对输入的数据进行反转义，检查密码和当前用户匹配后，再检查两次输入的修改密码是否相同，若相同则修改密码，其中数据库的查询和修改都使用预处理的 SQL 语句。
