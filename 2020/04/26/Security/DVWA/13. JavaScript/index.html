JavaScript

JavaScript 脚本攻击

Low

源码解析

使用 md5 + rot13 处理字符串，主要使用 generate_token 这个函数。

漏洞利用

直接提交 success 显示 Invalid token ，打开浏览器的开发者工具可以直接看到页面中的 JavaScript 代码，对输入的字符串执行 generate_token() ；

首先将输入改为 success ，然后在开发者工具的 Console 中执行 generate_token() ；

发现隐藏的 token 发生改变，提交后显示成功。

Medium

源码解析

页面中包含 medium.js ；

生成隐藏 token 。

漏洞利用

查看网页源码，存在隐藏的 token ，这次生成 token 的代码在 medium.js 文件中；

类似的方法，先将输入改为 success ，然后在开发者工具的 Console 中执行 do_elsesomething("XX") ，提交后显示成功。

High

源码解析

页面中包含 high.js ；

对 JavaScript 进行混淆。

漏洞利用

隐藏的 token 使用 high.js 生成；

生成 token 的主要步骤在最后几个函数中：

  1. 执行 token_part_1("ABCD", 44)
