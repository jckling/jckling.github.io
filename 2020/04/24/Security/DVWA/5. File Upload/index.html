File Upload

文件上传

Low

源码解析

使用 basename 返回路径中的文件名部分，对上传的文件不进行经检查和过滤，直接将上传的文件移动到上传目录。

漏洞利用

上传 info.php 文件查看 PHP 配置信息，内容如下：

上传成功后还返回文件的保存路径；

用这个路径查看并执行上传的文件。

参阅

  • PHP Manual - $_POST
  • PHP Manual - basename
  • PHP Manual - $_FILES
  • PHP Manual - move_uploaded_file

Medium

源码解析

利用 $_FILES 读取文件名称、类型、大小，限制格式为 jpeg 或 png ，并且大小必须小于 100000 字节。

漏洞利用

首先把 DVWA 的 hackable/uploads 目录还原，即在服务器的 /var/www/html/hackable/uploads 文件夹清空；然后把 info.php 改成 info.png 上传，利用 Burp Suite 抓包，将文件名修改为 info.php ；

结果显示为 info.php 上传成功；

根据反馈信息查看上传的文件，可以看到成功上传和执行了 info.php；从 Burp Suite 上可以看到对参数进行了修改，将 .png 改为 .php。

High

源码解析

用 strrpos 查找 . 在文件名中出现的最后位置，将前面的作为文件名，后面的作为文件扩展名；扩展名限制为 jpg、 jpeg、png，且大小必须小于 100000 字节；使用 getimagesize 获取图像文件的大小，判断文件格式是否为图像。

漏洞利用

首先将 info.php 拼接到一张图片的尾部（懒得改文件头）；

然后将其上传，显示 fake.jpg 成功上传；

查看上传的图片，显然没有被当作 .php 文件解析；

利用命令注入（Command Injection）查看当前路径并修改文件名， High 级别使用管道符 |；

查看修改后的文件，前面是图片信息，后面是执行 .php 的输出，显示 PHP 配置信息。

参阅

  • PHP Manual - substr
  • PHP Manual - strrpos
  • PHP Manual - getimagesize
  • DVWA FILE UPLOAD (HIGH LEVEL)

Impossioble

源码解析

使用 Anti-CSRF Token 来避免 CSRF 攻击，提交数据后先对 token 进行检查；然后依次读取文件的文件名、大小、类型，将 uniqid + 文件名 作为 md5 的输入，生成新的文件名；使用 getimagesize 获取图像文件的大小，判断文件格式是否为图像；保存文件时先使用 imagecreatefromjpeg 或 imagecreatefrompng 创建一个新图象，再使用 imagejpeg 或 imagepng 将其输出到临时图像文件中，最后用 imagedestroy 销毁图像的内存；将保存的临时图像文件重命名，并使用 file_exists 检查是还否存在临时图像文件，若存在则使用 unlink 删除。

参阅

  • PHP Manual - uniqid
  • PHP Manual - ini_get
  • PHP Manual - imagecreatefromjpeg
  • PHP Manual - imagejpeg
  • PHP Manual - imagecreatefrompng
  • PHP Manual - imagepng
  • PHP Manual - imagedestroy
  • PHP Manual - file_exists
  • PHP Manual - unlink
